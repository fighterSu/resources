# 社交平台

## 需求分析

### 区分用户

检测登录状态（过滤器、Session）->区分管理员和普通用户（用户标识符）

用户表（用户名、密码、用户id、超级管理员标识符）------ 登录时使用



### 注册用户

- 用户信息表（用户id，详细资料）---- 登录成功后，显示相关信息

- 分享内容表（内容id，分享者id，图片链接，文字（varchar(300)，个人屏蔽标识符，超级管理员屏蔽标识符) ----- 显示分享内容信息
- 点赞表(内容id，点赞者id)  ---- 查看点赞用户，用户查看自己点赞的内容
- 评论表(评论id，分享内容id，评论者id，评论内容) ---- 显示分享内容的评论
- 回复表(回复id，回复者id，回复的评论id，回复内容) ---- 显示对评论的回复
- 收藏表(用户id，内容id) ----查看自己的收藏内容



**这样可能效率有点低，可以将自己的评论和收藏之类的信息再建一个表**

收藏、评论、回复、点赞表（id，内容id，类型）



### 管理员

- 用户表(id，用户id，屏蔽标识符)
- 屏蔽关键字表（id，屏蔽关键字）
